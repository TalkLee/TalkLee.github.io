同名标识符优先级讨论

  • JavaScript语法

局部变量先使用后声明，不影响外部变量

var foo = 1;
function fn(){
    alert(foo);   // undefined 
    var foo = 2;
}
fn();

解析:
局部变量声明提前，即是外部变量有值，仍然按照作用域链顺序，获取到局部变量后返回其值

形参优先级高于函数名

function foo(foo) {
    alert(foo);   //1
}
foo(1);

解析:
JS解析foo为形参，故输出1，否则输出整个函数

形参优先级高于arguments

function foo(arguments) {
    alert(arguments); //1
}
foo(1,2);

解析:
JS解析arguments为形参，而非形参列表(arguments非保留字),否则输出[object Arguments]

初始化的函数内局部变量 > 形参优先级 > 未于初始化过的局部变量

1.

function fn(foo) {
    var foo = 1;
    alert(foo);   //1
}
fn(2);

2.

function fn(foo) {
    alert(foo);
    var foo = 1;  //2
}
fn(2);

3.

function fn(foo) {
    var foo;
    alert(foo);   //2
}
fn(2);





function fn(a) {
    var a = a;
    alert(a);
}
fn('hello');

解析:

该例中，表达式 var a = a ，右式a为形参，左式a为变量，所以输出为hello
