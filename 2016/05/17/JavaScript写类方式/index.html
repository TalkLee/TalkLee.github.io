JavaScript写类方式

  • 每日一博

本文讨论面向对象中类的书写方式,暂不考虑类的继承和私有方法属性

构造函数方法

function Person(name) {
    this.name = name;
    this.getName = function(){
        return this.name;
}

根据参数不同，构造拥有不同属性的对象实例
对象实例的方法每次都会被创建,浪费资源，例如getName这种方法完全可以所有同类型实例对象共享

改进，外部函数作为共享方法，所有对象的对应共享方法引用指向对象属性,通过对象属性实现对共享方法的调用

function getName() {
    return this.name;
}
function Person(name) {
    this.name = name;
    this.getName = getName;
}

原型方式

所有对象实例的属性均为共用，无法通过参数对对象实例属性进行定制

function Person() {}  //作为构造函数调用时，返回一个默认对象,包含Object原型对象的所有属性
Person.prototype.name = 'TalkLee';
Person.prototype.getName = function(){
    return this.name;
};

构造函数+原型
原型：用于生成共享方法
构造函数:用于定制拥有不同对象的对象实例

function Person(name) {
    this.name = name;
}
Person.prototype.getName = function(){
    return this.name;
};

结构上共享方法和对象实例属性关系不够清晰
将共享方法放置在构造函数中定义，清晰分辨出共享方法的性质

function Person(name) {
    this.name = name;
    Person.prototype.getName = function() {
        return this.name;
    }
}

但每次生成对象实例时都需要生成匿名函数，并且传递引用

故加入判断

function Person(name) {
    this.name = name;
    if(Person.hasInit == undefined ) {   //第一次生成对象实例
        Person.prototype.getName = function(){
            return this.name;
        }
    }
    Person.hasInit = 1;  //定义以后给对象标志属性赋值,以后即可用此对象进行判断
}

更新于2016.5.17

function $class(constructor,prototype) {   //生成类函数
    var c = constructor || function(){};
    var p = prototype || {}; 
    //c.prototype = p; 
    //构造函数原型指向传入prototype参数,导致构造器被改写(没看懂....)
    for(var i in p) {
        c.prototype[i] = p[i];  //将传入的原型对象属性遍历复制给构造函数的原型，防止对象字面量重写对象原型
    }
    return c;   //返回构造函数
}

以上函数用于传递构造函数和原型从而生成构造函数
通过生成的构造函数可以创建对象实例

更新于2016.5.18

function $class(constructor,prototype) {
    var c = constructor || function(){};
    var p = prototype || {};
    return function() {
        for(var i in p) {
            //匿名函数原型获取来自传入的原型的属性,相当于不断给prototype对象添加新属性，每一个生成的构造函数即使原型的属性相同，但是也是不同的对象
            arguments.callee.prototype[atr] = p[atr];
        }
        //构造函数产生的新对象中调用构造函数
        c.apply(this,arguments);
    }
}

对之前的生成类的函数做了优化，生成的构造函数原型不指向同一对象.

更新于2016.5.19

function $class(className,superClass,classImp) {
    //超类默认为Object类型
    if(superClass === '') {
            superClass = Object;
    }

    function clazz() {
        if(typeof this.init == 'function') {
            this.init.apply(this,arguments);
        }
    }
    //clazz构造函数的原型指向超类
    var p = clazz.prototype = new superClass();
    //超类指向 _super
    var _super = superClass.prototype;
    //自定义对象生成为全局内置对象
    window[className] = clazz;
    classImp.apply(p,[_super]);
}

clazz和classImp没看懂
使用该工具函数写类需注意，this.init方法必不可少。使用过Prototype库的同学会知道Class.create后的initialize方法也是必不可少的。
