for in方法缺陷(标准中已修复)

  • 每日一博
var obj = {
    foo : 19,
    toString : function(){
        alert('new');
    }
};
for(var i in obj) {
    alert(i);
}

该例中for in既输出了自定义属性foo，又输出重写的toString.
测试后发现，IE5以上版本均与上述行为相同。

给上例添加下述代码:

Object.prototype.kiss = function(){
};

kiss方法同样输出

得出结论 :
for in可以输出用户自定义的属性以及添加在声明对象原型上的属性，即使是重写的内置属性，如该例中toString方法.

不足之处:

for in既可以获取对象自有属性，也可以获取从原型中共享得到的属性。需要通过hasOwnProperty方法判断自有属性
