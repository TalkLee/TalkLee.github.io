sass review

  • sass

命令行编译:

sass main.scss // 指定scss文件,编译为css(本文使用sass格式)

变量:

$nav-color : #123;  //定义变量,用于之后的属性值复用,值可以是任何CSS属性值,包括复合属性值

变量值输出时根据之前最近一次的初始化计算.

作用域:

块作用域, {}中为局部变量，只能作用于该规则块中。
{}外则可作用在整个样式表中

变量未定义时的默认值: 若用户未定义变量值，则使用默认值，否则使用用户定义值

$x : 1,
$x : 5 !default;  //$x = 1
$y : 3 #default;  //$y = 3

变量名中多个词之间使用中划线或者下划线均可.

Sass中命名的地方，中划线等价下划线，但是原生CSS命名不互通.

变量作用域:

全局变量后加 !global , 否则变量具有嵌套作用域

嵌套:

选择器嵌套:

.container{
    .nav_bar{
        ....            //实现多级选择器
    }
}

内部选择器若需要引用父级选择器使用标记&,即是父选择器引用,效果不变
引用CSS规则集中的根选择器@at-root.

群组选择器的嵌套:
按照顺序依次和外部选择器连接:

.container{
    h1,h2,h3{
        border : 1px solid;
    }
}
等价的css写法：
.container h1, .container h2, .container h3{border : 1px solid;}

属性嵌套(和选择器嵌套区别在于主体属性后面跟冒号:):

复合属性 = 主题属性 - 具体属性 

.container{
    font:{   //注意后面的冒号: 
        family : sans-serif;
        weight : bold;
    }
}

选择器插值:

@prefix : ui ;
.#{$prefix}-button {  //解析为.ui-button
    ...
}

@import 语句插值

@import ulr(styles.#{$device}.css);

属性名插值:

border-{side}-color

@import写法：

混合器(mixin),复用CSS规则集 :

@mixin large-text {
    font : {
        family : sans-serif;
        weight : bold;
    }
    color : #123;
}
.page-title {
    @include large-text;  //引入之前定义的large-text CSS规则集
}

混合器和CSS中类名规则集区别:

CSS中类名表示HTML元素的语义，Sass中混合器直接作用于样式，表示的是样式描述

混合器中还可以包括CSS规则集(指定元素的样式声明):

@mixin large-text {
    color:#123;
    li{     //
        list-style:none;
        margin : 0;
    }
}

混合器中还支持参数:

@nav-border($width : 10px, $color : #123,$height) {  //可以设置参数默认值
    border-width : $width;
    border-color :$color;
    height : $height;
} 
footer {
    @inlcude nav-border(1px,#132);  //传入参数
}

混合器用于样式层面的复用

选择器继承可以防止像混合器一样产生属性重复

.err {
    border : 1px solid;
    background:#123;
}
.badError{
    @extend .err ; //继承err,复用err的所有样式
9
}

混合器复用属性，选择器继承复用选择器,类似子类之于父类的关系，继承父类(元素)的样式

混合器的工作细节:将引入的混合器样式规则复制到引入的规则集中。

@mixin test {
    border :1px solid #000;
}
span {
    @include test;
}

等价于:

span {
    border :1px solid #000;
}

继承的工作细节:找到父类选择器的规则集，然后将选择器替换为父类 + 子类的形式:
只要存在父类选择器的规则集，就一定将该规则集运用到子类上。父类子类说法只是便于理解，实际可以是任何选择器

div.test{
    border :1px solid;
}

span {
    @extend div.test;
}

等价于:
div.test span{
border:1px solid;
}

逻辑控制:

  • @if
  • @else if
  • @else
  • @for
  • @each
  • @while

Sass 文件导入(@import):

原生CSS中@import规则允许在CSS文件中导入其他CSS文件,当运行到@import规则时才下载对应CSS文件。

Sass中@import规则则将引入的css文件合并后才编译。生成单一CSS样式文件

@import规则默认在不书写引入样式文件的后缀时，默认有限搜索sass及scss文件

Sass局部文件:

作为复用组件的Sass文件，只用于在其他多个Sass文件中被导入，无需编译生成对应css文件。
对于此类文件,以下划线开头，Sass就会识别为局部文件，从而不对此文件编译，而只用于导入，同时，输入文件名的时候，文件名也可以省略首字母下划线和sass或scss后缀.

@import 'src/sass/component/header'  //用于引用css组件header,该文件名为_header.scss

Sass文件也可以导入到CSS规则集中

header {
    @import 'src/sass/component/header'
}

以下三种情况下使用原生CSS导入:

  • 导入文件以.css文件为后缀
  • 导入文件为URL地址(http://localhost:63342/task9/index.html/src/header.css)
  • 导入文件为css的URL值

注释:

// 单行注释,注释内容不出现在编译后的css文件中
/* */ 多行注释，注释内容出现在编译后的css文件中
