gulp上手

  • gulp
  • 前端工具

gulp配置文件(gulpfile.js)书写介绍:

配置之前需要先安装gulp插件

 var gulp = require('gulp');  //加载gulp及其插件模块

gulp.task(taskName, function() {
    //task具体执行的行为
});
gulp.task('default',function() {
    //设置default任务
    gulp.run(task1,task2,taskn); //设置默认任务需要执行的其他子任务
    gulp.watch(filename,function() {  //监听指定文件(路径及其文件名)变化，变化时调用回调函数,执行指定任务
        gulp.run(task1,task2,taskn);
        })
    })

配置文件设置完毕,

命令行使用gulp命令运行gulp.js构建程序运行default任务,或者gulp + taskName 运行指定任务

常用API：

gulp.task(name[,deps],fn) :
name : 任务名
deps : 当前定义任务执行需要依赖的其他任务(数组),效果是依赖任务执行完以后才执行此时定义的任务,可选
fn : 任务执行的具体代码,可选

所有定义的任务之间如果没有依赖，任务会按照定义顺序以此执行，
但是若依赖的任务是中包含异步操作,依赖主体并不会等到依赖任务的异步操作完成后才执行.

如果task中为异步任务,需要通过调用一个回调函数或者返回一个event stream或promise
来提示任务已经完成(写在fn的最后面)

gulp.watch(glob[,opts],tasks | cb ) :

glob : 监听的文件(包括路径、后缀)或文件数组
tasks : 变化时执行指定任务(数组)
cb : 变化时执行的回调函数,传入一个event对象，包括type和path两个属性,表示发生改变的类型(增删改)，触发事件的文件路径

gulp.dest(path[,options])
写入文件
path : 导出文件的路径
options : 可选参数对象,不常用
将pipe的文件写入到指定目录下的文件中,文件夹若不存在则创建之
path只能指定导出文件的路径，文件名是由导入到文件流自身的文件名,即生成的文件名是由导入的到该方法的文件流决定的,

gulp.src(globs[,options])
读取文件到流
options : 可选参数对象,不常用
globs:待读取文件的匹配模式(数组),类似正则表达式匹配,写法参考node-glob写法

gulp运行流程：
先通过src方法返回一个Vinyl file类型的stream,可以通过pipe方法将stream导入到gulp的插件,之后将经过插件处理后的流通过pipe方法导入到gulp.dest()中，dest方法将流生成为导出文件,除此之外还可以将插件处理后的流导入到其他插件中继续处理

生成的文件路径为dest方法中传入的path路径加上src方法中glob匹配模式中通配符出现后的那部分路径 d，而文件名则由流决定

明日更新dest等方法搭配产生的效果

Vinyl

vinyl是一种非常简单的用于描述文件的元数据对象。当我们形容一个文件时,必然包含两个属性:

  • path 路径
  • contents 内容
    这也是vinyl的主要属性,
    除此之外vinyl并不仅仅描述你电脑文件系统中的文件，还可以是云端上的文件。

Vinyl 适配器

vinyl提供了一种简单的表述文件的方式,但我们还需要一种方式去获取这些vinyl表述的文件,每种文件资源都需要vinyl适配器的存在。而vinyl适配器简单暴露了 src和 dest方法，以上的两个方法均返回一个stream,src方法返回的流(简称src流)产生Vinyl对象，而dest流则消费Vinyl对象。除此以外，Vinyl适配器还暴露了其他一些方法特定与他们的input/output 媒介，例如vinyl-fs提供的symlink方法
