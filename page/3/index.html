判断对象是否具有指定属性

两种方式:

in运算符，判断是否为指定对象的属性，实例属性或从原型共享的属性
hasOwnProperty方法 判断是否为制定对象的实例属性

var obj = {
    foo : '123',
    toString : function () {

    }
};
alert('asd' in obj); //true
alert(obj.hasOwnProperty('asd'));  //true 
alert('toString' in obj);  //true
alert(obj.hasOwnProperty('toString'));  //true

for in 无法检测原型链上的属性.
只能输出用户添加在对象和添加在对象原型上的属性，不输出JS内置属性.

for in方法缺陷(标准中已修复) 

var obj = {
    foo : 19,
    toString : function(){
        alert('new');
    }
};
for(var i in obj) {
    alert(i);
}

该例中for in既输出了自定义属性foo，又输出重写的toString.
测试后发现，IE5以上版本均与上述行为相同。

给上例添加下述代码:

Object.prototype.kiss = function(){
};

kiss方法同样输出

得出结论 :
for in可以输出用户自定义的属性以及添加在声明对象原型上的属性，即使是重写的内置属性，如该例中toString方法.

不足之处:

for in既可以获取对象自有属性，也可以获取从原型中共享得到的属性。需要通过hasOwnProperty方法判断自有属性

sass review

命令行编译:

sass main.scss // 指定scss文件,编译为css(本文使用sass格式)

变量:

$nav-color : #123;  //定义变量,用于之后的属性值复用,值可以是任何CSS属性值,包括复合属性值

变量值输出时根据之前最近一次的初始化计算.

作用域:

块作用域, {}中为局部变量，只能作用于该规则块中。
{}外则可作用在整个样式表中

变量未定义时的默认值: 若用户未定义变量值，则使用默认值，否则使用用户定义值

$x : 1,
$x : 5 !default;  //$x = 1
$y : 3 #default;  //$y = 3

变量名中多个词之间使用中划线或者下划线均可.

Sass中命名的地方，中划线等价下划线，但是原生CSS命名不互通.

变量作用域:

全局变量后加 !global , 否则变量具有嵌套作用域

gulp上手

gulp配置文件(gulpfile.js)书写介绍:

配置之前需要先安装gulp插件

 var gulp = require('gulp');  //加载gulp及其插件模块

gulp.task(taskName, function() {
    //task具体执行的行为
});
gulp.task('default',function() {
    //设置default任务
    gulp.run(task1,task2,taskn); //设置默认任务需要执行的其他子任务
    gulp.watch(filename,function() {  //监听指定文件(路径及其文件名)变化，变化时调用回调函数,执行指定任务
        gulp.run(task1,task2,taskn);
        })
    })

配置文件设置完毕,

命令行使用gulp命令运行gulp.js构建程序运行default任务,或者gulp + taskName 运行指定任务

常用API：

gulp.task(name[,deps],fn) :
name : 任务名
deps : 当前定义任务执行需要依赖的其他任务(数组),效果是依赖任务执行完以后才执行此时定义的任务,可选
fn : 任务执行的具体代码,可选

所有定义的任务之间如果没有依赖，任务会按照定义顺序以此执行，
但是若依赖的任务是中包含异步操作,依赖主体并不会等到依赖任务的异步操作完成后才执行.

如果task中为异步任务,需要通过调用一个回调函数或者返回一个event stream或promise
来提示任务已经完成(写在fn的最后面)

gulp.watch(glob[,opts],tasks | cb ) :

glob : 监听的文件(包括路径、后缀)或文件数组
tasks : 变化时执行指定任务(数组)
cb : 变化时执行的回调函数,传入一个event对象，包括type和path两个属性,表示发生改变的类型(增删改)，触发事件的文件路径

gulp.dest(path[,options])
写入文件
path : 导出文件的路径
options : 可选参数对象,不常用
将pipe的文件写入到指定目录下的文件中,文件夹若不存在则创建之
path只能指定导出文件的路径，文件名是由导入到文件流自身的文件名,即生成的文件名是由导入的到该方法的文件流决定的,

gulp.src(globs[,options])
读取文件到流
options : 可选参数对象,不常用
globs:待读取文件的匹配模式(数组),类似正则表达式匹配,写法参考node-glob写法

gulp运行流程：
先通过src方法返回一个Vinyl file类型的stream,可以通过pipe方法将stream导入到gulp的插件,之后将经过插件处理后的流通过pipe方法导入到gulp.dest()中，dest方法将流生成为导出文件,除此之外还可以将插件处理后的流导入到其他插件中继续处理

生成的文件路径为dest方法中传入的path路径加上src方法中glob匹配模式中通配符出现后的那部分路径 d，而文件名则由流决定

###明日更新dest等方法搭配产生的效果

####Vinyl

vinyl是一种非常简单的用于描述文件的元数据对象。当我们形容一个文件时,必然包含两个属性:

  • path 路径
  • contents 内容
    这也是vinyl的主要属性,
    除此之外vinyl并不仅仅描述你电脑文件系统中的文件，还可以是云端上的文件。

####Vinyl 适配器

vinyl提供了一种简单的表述文件的方式,但我们还需要一种方式去获取这些vinyl表述的文件,每种文件资源都需要vinyl适配器的存在。而vinyl适配器简单暴露了 src和 dest方法，以上的两个方法均返回一个stream,src方法返回的流(简称src流)产生Vinyl对象，而dest流则消费Vinyl对象。除此以外，Vinyl适配器还暴露了其他一些方法特定与他们的input/output 媒介，例如vinyl-fs提供的symlink方法

ECMAscript数组迭代方法和归并方法

###ECMAscript5 数组 迭代方法

迭代方法(IE9+才支持)

接收参数:

  1. 数组每项上运行的函数,函数接受三个参数:
    1. 数组当前项的值
    2. 该项在数组中的下标
    3. 数组对象本身
  2. 运行该函数的作用域对象

迭代方法统计:

var arr = [];
var every = arr.every(function(item,index,array){
    //every,如果传入函数对每一项返回值均为true,返回true
});
var filter = arr.filter(function(item,index,array){
    //filer,返回由每次在数组元素上调用传入函数返回的布尔值为true的元素组成的数组
});
var forEach = arr.forEach(function(item,index,array){
    //forEach,对每项进行操作，无返回值 
});
var map = arr.map(function(item,index,array){
    //map,对每项进行操作,返回每次调用函数的结果组成的数组 
});
var some = arr.some(function(item,index,array){
    //some,对每项进行操作，如果存在至少一项函数调用后返回值为true,则返回true 
});

观察可以发现:

every 和 some 用于对整个数组做逻辑判断:

  • every表示任意项成立,则数组对指定条件成立
  • some表示存在一项成立，则数组对指定条件成立

map 和 foeEach 用于遍历数组进行操作,区别在于:

  • forEach 应用场景不需要返回值
  • map 对数组操作的返回值由需求

filter则用于对数组元素进行某种条件的筛选，通过返回值的不同对数组元素进行区别

归并方法(IE9+才支持):

  1. reduce
  2. reduceRight

区别:
reduce从前往后迭代,构建最终的值
reduceRight从后往前迭代

接收参数如下:

  1. 每一项上调用的函数,该函数接受接收4个参数:
    1. 前一个值
    2. 当前值
    3. 当前项索引
    4. 数组对象
      该函数返回的值作为在下一项上的迭代基础值，即函数第一个参数传入到下一项的函数中
  2. 作为归并基础的初始值

例:

var values = [1,2,3,4,5];
var sum = values.reduce(function(prev,cur,index,array){
    return prev + cur;
});
alert(sum);  //15

ejs入门

EJS简要介绍:
EJS是JS模版引擎的一种，风格类似ERB,应用于客户端，目前Node.js中也可以使用了。

主要有三种语法:
<% JS代码 %> :在<% %>之间可以执行JS代码
<%= %> : 对之间的代码进行HTML转义
<%- %> : 不对之间代码进行转义(例如：&字符)

客户端使用方法如下:

1.新建ejs文件，然后书写模版文件
2.在html文件中添加JS代码如下:

html = new EJS( {url : 'template.ejs'} ).render(data);   //url为模板文件所在路径,data为需要填充到模版文件的数据
document.body.innerHTML = html;  //将渲染函数产生的html标签及其结构插入到body中
//注: 该例中模版文件未书写<html>等标签

总结:

掌握这些就可以完成简单的数据填充任务了,个人也觉得这些只是就完全够用了，从项目用途来说，模板引擎仅仅只是用来做数据填充而已,EJS已经足够胜任了

###变量命名和对象的属性命名区别

变量命名要求:

第一个字母必须是字母、下划线、美元符号,但数字不能作为首字母

对象属性读取方式:

  2. 下标法[]

点取法中点号后面的标识符要求是合法标识符(符合合法的变量命名规则)
下标法中[]内只要是字符串即可,即使不符合变量命名规则

对象的键的类型为String,

.和[]法获取对象属性 

var obj = {
    a : 1,
    b : 2
};
alert(obj.a);  // 1
alert(obj['a']);  // 1
alert(obj[a]); // Uncaught ReferenceError : a is not defined

分析如下:

点取法将后面标识符解析为属性名
下标法中下标为字符串，匹配属性名
下表法中如果下标非字符串，会解析为变量，由于之前未定义变量，便会报错
