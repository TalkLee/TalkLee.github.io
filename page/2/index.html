类数组转化为数组

类数组: arguments 等，

  • 具有length属性，
    • 按照索引方式存储数据
    • 不具有push和pop方法(但是如果对象及其length属性为可读写属性,即可调用数组的push和pop方法)

可以直接在类数组对象上直接调用数组原型的方法:

Array.prototype.push.call(obj,newElement)

需要作为真实数组多次处理时，须转化为数组:

var arr = Array.prototype.slice.call(obj,0);

建议封装到个人常用函数库进行复用.

测试如下:

var obj = {};
obj[0] = '1';
obj[1] = '2';
obj[2] = '3';  
obj.length = 3;  //模拟类数组，即只需要满足开头所说要求，即可视为类数组
alert(arr instanceof Array);  //true
alert(obj instanceof Object);  //true
alert(arr[1]);     //'2'
alert(obj['1']);   //'2'

IE8及以下存在兼容性问题，
需要显式设置类数组的length属性(slice中使用)，否则上述测试中 alert(arr[1]) 返回undefined

attribute和property区别

MDN翻译如下:

attribute拓展了标签，给标签提供了新的行为或者元数据。attribute总是有着一个键值对的表格，赋予attribute标识符以及相关联的值。

####property：

DOM对象的property值通过点取法获取

document.body.innerHTML;

DOM对象作为对象，可以在脚本中自定义property，而自定义property不出现在html文档中
所以，property标识符定义只遵守js标识符命名规范

####attribute:

DOM节点提供以下方法访问attribute(存在兼容问题)

  • hasAttribute
  • getAttribute
  • setAttribute
  • removeAttribute

由于是DOM节点的“属性”，所以遵守HTML标识符规范：

  • 值只能为字符串
  • 不区分大小写
  • html中出现
  • 可以通过DOM对象的属性:attributes对象获取所有attribute组成的一个类数组NamedNodeMap
  • 其实attribute键名也为字符串形式_

JS中操作DOM对象的property只需要点取法或者下标法即可获取，而除了‘共通属性’外的其他attribute，则只能通过JS自带attribute方法获取。html自定义属性(自定义attribute)也能通过attribute一类方法访问

常用attriubte中，id、class、title、style等也作为property附加到DOM对象中了

####特殊‘共通属性’分辨:

class:

HTML中class作为attribute，在getAttribute等方法中，应该传入’class’(字符串形式)
作为property，由于class为保留字，所以改为className

alert(test['class']);  // 尝试通过property方法获取,失败
alert(test['className']); //property中class正确书写为className

style:

alert(test.style);    //返回div1指向的DOM节点的CSSStyleDeclaration对象
alert(test.getAttribute('style'));  //返回一个包裹了对应DOM节点的样式 字符串

PS：attribute方法返回的是行内样式字符串

onclick:

alert(typeof btn.getAttribute('onclick'));  //string
alert(typeof btn.onclick);   //function

个人建议:
当试图将‘属性’看作attribute访问时，最好通过JS自带 attribute 方法进行访问，点取法和下标法用于获取property，易读，
若考虑兼容性，可对特定‘共通属性’作规范.

同名标识符优先级讨论

####局部变量先使用后声明，不影响外部变量

var foo = 1;
function fn(){
    alert(foo);   // undefined 
    var foo = 2;
}
fn();

解析:
局部变量声明提前，即是外部变量有值，仍然按照作用域链顺序，获取到局部变量后返回其值

####形参优先级高于函数名

function foo(foo) {
    alert(foo);   //1
}
foo(1);

解析:
JS解析foo为形参，故输出1，否则输出整个函数

####形参优先级高于arguments

function foo(arguments) {
    alert(arguments); //1
}
foo(1,2);

解析:
JS解析arguments为形参，而非形参列表(arguments非保留字),否则输出[object Arguments]

初始化的函数内局部变量 > 形参优先级 > 未于初始化过的局部变量

1.

function fn(foo) {
    var foo = 1;
    alert(foo);   //1
}
fn(2);

2.

function fn(foo) {
    alert(foo);
    var foo = 1;  //2
}
fn(2);

3.

function fn(foo) {
    var foo;
    alert(foo);   //2
}
fn(2);





function fn(a) {
    var a = a;
    alert(a);
}
fn('hello');

解析:

该例中，表达式 var a = a ，右式a为形参，左式a为变量，所以输出为hello

JavaScript写类方式

本文讨论面向对象中类的书写方式,暂不考虑类的继承和私有方法属性

####构造函数方法

function Person(name) {
    this.name = name;
    this.getName = function(){
        return this.name;
}

根据参数不同，构造拥有不同属性的对象实例
对象实例的方法每次都会被创建,浪费资源，例如getName这种方法完全可以所有同类型实例对象共享

改进，外部函数作为共享方法，所有对象的对应共享方法引用指向对象属性,通过对象属性实现对共享方法的调用

function getName() {
    return this.name;
}
function Person(name) {
    this.name = name;
    this.getName = getName;
}

####原型方式

所有对象实例的属性均为共用，无法通过参数对对象实例属性进行定制

function Person() {}  //作为构造函数调用时，返回一个默认对象,包含Object原型对象的所有属性
Person.prototype.name = 'TalkLee';
Person.prototype.getName = function(){
    return this.name;
};

构造函数+原型
原型：用于生成共享方法
构造函数:用于定制拥有不同对象的对象实例

function Person(name) {
    this.name = name;
}
Person.prototype.getName = function(){
    return this.name;
};

结构上共享方法和对象实例属性关系不够清晰
将共享方法放置在构造函数中定义，清晰分辨出共享方法的性质

function Person(name) {
    this.name = name;
    Person.prototype.getName = function() {
        return this.name;
    }
}

但每次生成对象实例时都需要生成匿名函数，并且传递引用

故加入判断

function Person(name) {
    this.name = name;
    if(Person.hasInit == undefined ) {   //第一次生成对象实例
        Person.prototype.getName = function(){
            return this.name;
        }
    }
    Person.hasInit = 1;  //定义以后给对象标志属性赋值,以后即可用此对象进行判断
}

更新于2016.5.17

function $class(constructor,prototype) {   //生成类函数
    var c = constructor || function(){};
    var p = prototype || {}; 
    //c.prototype = p; 
    //构造函数原型指向传入prototype参数,导致构造器被改写(没看懂....)
    for(var i in p) {
        c.prototype[i] = p[i];  //将传入的原型对象属性遍历复制给构造函数的原型，防止对象字面量重写对象原型
    }
    return c;   //返回构造函数
}

以上函数用于传递构造函数和原型从而生成构造函数
通过生成的构造函数可以创建对象实例

更新于2016.5.18

function $class(constructor,prototype) {
    var c = constructor || function(){};
    var p = prototype || {};
    return function() {
        for(var i in p) {
            //匿名函数原型获取来自传入的原型的属性,相当于不断给prototype对象添加新属性，每一个生成的构造函数即使原型的属性相同，但是也是不同的对象
            arguments.callee.prototype[atr] = p[atr];
        }
        //构造函数产生的新对象中调用构造函数

对之前的生成类的函数做了优化，生成的构造函数原型不指向同一对象.

更新于2016.5.19

function $class(className,superClass,classImp) {
    //超类默认为Object类型
    if(superClass === '') {
            superClass = Object;
    }

    function clazz() {
        if(typeof this.init == 'function') {
            this.init.apply(this,arguments);
        }
    }
    //clazz构造函数的原型指向超类
    var p = clazz.prototype = new superClass();
    //超类指向 _super
    var _super = superClass.prototype;
    //自定义对象生成为全局内置对象
    window[className] = clazz;
    classImp.apply(p,[_super]);
}

使用该工具函数写类需注意，this.init方法必不可少。使用过Prototype库的同学会知道Class.create后的initialize方法也是必不可少的。

具名函数的多种调用方式

背景代码:

function foo(){
    alert('a');
}

方法1:

括号()运算符调用

foo();

方法2&3:

call和apply方法

foo.call();  //call和apply中传入的第一个参数为函数调用中作用域(this对象),
单纯为调用函数的话，传入变量常量任何均可
foo.apply();

方法4：

new构造函数调用方法

new foo()

注意以下情况:

    function foo(param){
    alert(param);
    return param;   //有返回值
}

var a = new foo({ b : '1'});  //返回传入对象
var a = new foo('OK');    //返回另一个对象

若函数声明中返回对象，则构造函数调用时返回指定的对象
若函数声明中返回基本类型(包括null和undefined),构造函数调用时实际返回函数语句中生成的对象(通过this给生成的对象添加属性)

function foo(param){
    alert(param);
    this.b = '1';
    return null;
}

上例中return语句无效,返回对象，包含属性b.

若函数声明中声明新对象，并且通过this给构造函数生成对象添加属性，又会出现什么情况呢?

若无返回值或者返回基本类型或者返回this，则返回this产生的对象，
若返回构造函数中声明的对象，则返回指定的对象(工厂模式的写法)

function foo(param){
    this.pro = 'this';
    var obj = {};
    obj.name = 'foo';
    obj.msg = param;
    //return this;  返回对象中包括pro属性
    //return obj;  返回对象为obj
}

工厂模式:

function foo(param){
    var obj = {};
    obj.msg = param;
    return obj;
}
var a = new foo('2');  //构造函数方式调用
var b = foo('3');  //函数调用,两种方法返回结果相同，原因上面已经讨论过了

遍历数组

严格意义上来说，这篇博客更应该叫数组元素获取的几种方法:

常用方法:

(for var i = 0 ;i < array.length ; i++) {
    array[i]  
}

方法2：

for(var a in array){
    console.log(a); //返回对应元素的下标,即下标即可看作元素作为数组对象的属性
    console.log(array[a]);  
}

由于JS中数组实际也是Object，所以，遍历对象属性的方法同样适用于数组，但性能太差

点取法:

console.log(array.a); //报错，因为数组(对象)中不包括a属性

下标法:

console.log(array[a]); //中括号中a被转化为字符串类型，等价于array['0']，所以正确

即，中括号中标识符值被转化为字符串类型

var a = 1;
console.log(array[a]);

该例中，a = 1;所以array[a] 等价于 array[‘1’];

####对象中的键均为字符串类型

方法3:

for(var i=0,a;a=ary[i++];){  //比较好奇为什么可以不用加终止条件
    alert(i);
    console.log(a);
}

方法4:

for(var i=0,a;a=["jack","tom","lily","andy"][i++];){  //每次都创建一个数组常量，性能问题
    console.log(a);

}

判断对象是否具有指定属性

两种方式:

in运算符，判断是否为指定对象的属性，实例属性或从原型共享的属性
hasOwnProperty方法 判断是否为制定对象的实例属性

var obj = {
    foo : '123',
    toString : function () {

    }
};
alert('asd' in obj); //true
alert(obj.hasOwnProperty('asd'));  //true 
alert('toString' in obj);  //true
alert(obj.hasOwnProperty('toString'));  //true

for in 无法检测原型链上的属性.
只能输出用户添加在对象和添加在对象原型上的属性，不输出JS内置属性.
